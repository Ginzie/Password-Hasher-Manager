How this works:

You type a password, and a key. The password will change depending on the key
(You can have the same password for everything, but if the key is different
then the password is different). This program is entirely offline and
does not store passwords or keys locally or externally so either remember them
or write them down.

For the sake of transparency, the steps of the algorithm are as follows:

Step 1: Add up the key
  • Look at every letter in the key.
  • For each letter, find its number value (like A=65, B=66).
  • Add all those numbers together to get a starting total.

Step 2: Mix the password with the key total
  • Look at each letter in the password.
  • For each letter, turn it into its number value and multiply it by the current key total.
  • After each letter, lower the key total a little (subtract twice the length of the key).
  • Write down all these numbers one after another to make one big long string of digits.

Step 3: Break the long number string into pieces
  • Start with a “helper number” that’s the length of the password plus the length of the key.
  • Go through the long string of digits:
    • If the helper number is even, take the next two digits.
    • If it’s odd, take the next one digit.
  • After each chunk, update the helper number by multiplying it by 1.5 and rounding down.

Step 4: Turn numbers into safe letters and symbols
  • Take each chunk and add 32 to make it a normal printable character.
  • If that character is “unsafe” (like <,>, &, or anything weird), fix it:
    • Square the number and then shrink it down into the printable range.
    • If it’s still unsafe, just increase the number by 1.
    • Wrap around if it goes too high.
  • Convert the number to a real letter or symbol.

Step 5: Combine all the characters
  • Stick all the safe letters and symbols together.
  • That’s your final hashed password.
